# Лабораторная работа №5

## Тема

Топологическая сортировка ориентированного графа, заданного матрицей смежности.

## Постановка задачи

Необходимо реализовать структуру графа на матрице смежности и алгоритм топологической сортировки. Программа должна уметь добавлять и удалять рёбра, выполнять топологическую сортировку и определять ситуацию, когда сортировка невозможна из-за цикла.

## Реализация

Граф хранится в структуре `graph`:

- `matrix` - матрица смежности;
- `cnt_v` - текущее количество вершин;
- `capacity_v` - выделенная ёмкость матрицы.

Топологическая сортировка реализована алгоритмом Кана.

## Сборка и запуск

```bash
cmake -S . -B build
cmake --build build
```

Демо-программа:

```bash
./build/graph_demo
```
Запуск из файла:

```bash
./build/graph_demo input.txt output.txt
```
формат файла input.txt:
```terminaloutput
3
1
0
1
1
1

3
4
6

```

Чтобы записать граф на диск нужно выбрать 6 и он будет записан в graph.txt

Чтобы записать результат топологической сортировки нужно выбрать 4 и он будет записан в topsort.txt

Бенчмарк:

## Unit-тесты

Тесты написаны через GoogleTest. Проверяются:

- создание графа;
- добавление ребра;
- сортировка простого DAG;
- обнаружение цикла;
- более сложный DAG с несколькими допустимыми порядками.

Результат запуска:

```text
100% tests passed, 0 tests failed out of 5
```

## Benchmarking

Платформа:

- CPU: Apple M4
- RAM: 16.0 GB
- OS: macOS 15.6.1

Методика: для каждого размера создавался ориентированный ациклический граф на матрице смежности. Вершины нумеровались от `0` до `n - 1`. Для каждой вершины `i` добавлялись рёбра в следующие вершины `j`, где `i < j < i + 9`, то есть у одной вершины было не более 8 исходящих рёбер. Так как все рёбра идут только из вершины с меньшим номером в вершину с большим номером, граф не содержит циклов.

Перед измерением выполнялся один пробный запуск `topological_sort`. Затем сортировка запускалась 100 раз, время измерялось функцией `clock()` из `<time.h>`. В таблице указано среднее время одного запуска.

| Вершины | Среднее время одного запуска, мс |
|--------:|---------------------------------:|
| 100     | 0.044                            |
| 500     | 0.594                            |
| 1000    | 1.705                            |
| 2000    | 6.843                            |
| 4000    | 13.550                           |

## Вывод

Алгоритм корректно строит топологический порядок для DAG и возвращает ошибку при наличии цикла. Результаты бенчмарка соответствуют ожидаемой сложности `O(V^2)`: при увеличении числа вершин время растёт примерно квадратично, так как используется матрица смежности.
